(static) enterGalaxy(keyLocation, botInfo)

not implemented yet

ssb-client expects to have a scuttlebot running someplace it can reach. all interaction with it is through callbacks.

this function in sbot's case is just for spinning up that child process, if there isn't one already.

so we have three cases:

1) bot not running -> start with with above key location and return process ref

2) bot running, key location is different from running bot -> new identifier, so return a partial application (curry) of ssbClient that includes botInfo

3) bot running, key location is empty -> default identifier; return ok

for type/interface sanity this means we should return an object that describes which of these things happened, containing the return of above.

TODO replace all of this with something like RPC on scuttlebot-views
